What to Check on Your Server

Run this command on your backend server:

cat /path/to/nexa/backend/.env | grep APPLE

Expected output:

APPLE_BUNDLE_ID=com.pymesoft.nexa APPLE_SERVICE_ID=com.pymesoft.nexa.web

‚ö†Ô∏è If APPLE_SERVICE_ID is missing, that's why Apple Sign In fails!

How to Fix

  1. SSH into your backend server
  2. Edit /path/to/nexa/backend/.env
  3. Add: APPLE_SERVICE_ID=com.pymesoft.nexa.web
  4. Restart backend: pm2 restart all
  5. Test again on app.nexapymesoft.com
